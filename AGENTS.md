# OpenCode Agent Instructions

This repository integrates Obsidian, Anki, and Google Gemini into an automated PKM and multi-agent workflow.

## 🛠️ Key Developer Commands
- **Two-way Sync:** `python sync.py` (Requires local Anki running with AnkiConnect on port 8765).
- **Daily Note Splitter:** `python daily_to_atomic.py` (Scans `Daily Notes/` for end-of-note drafts, extracts them to `Atomic Notes/` with a UUID).
- **Semantic Related Notes:** `bash run_similarity.sh` (Shortcut that launches the user `qdrant` service, activates the venv at `/home/shioriko/scripts/Obsidian/venv`, and runs `similarity_manager.py`).
- **Homework Solver:** `python solve_hw.py <filepath>` (Requires external binaries `pandoc` and LibreOffice headless `soffice` for pptx).
- **Multi-Agent Routing:** `python router.py <note_path>` (Dispatches dynamically based on `@agent_name` in the last `> ` block).

## 🤖 Multi-Agent Architecture & Quirks
- **Gemini SDK Only:** Exclusively use the new Google GenAI SDK (`from google import genai` and `from google.genai import types`). NEVER use the legacy `google-generativeai`. Initialize clients with `api_key=utils.GOOGLE_API_KEY`.
- **Dynamic Routing:** All Python modules in `agents/*.py` (except `__init__.py`) containing a `handle(command, history, note_path, full_content)` function are automatically loaded.
- **Parsing Structure (`utils.py`):** Chat history uses markdown quote syntax (`> ` indicates user turn; other lines indicate assistant turn). `utils.parse_document(content)` extracts and returns a 5-tuple (`full_header, context_header, chat_content, related_body, cards_body`) to separate `## 相关笔记` from the conversation. The flashcard table is strictly expected under a `## 卡片` header at the very bottom of the note.
- **Draft-Confirm Protocol:** `@add` and `@revise` agents use a draft-confirm pattern. They first output a Pydantic-validated JSON draft table. They do NOT write to the note until a second invocation containing `ok`, `yes`, `y`, or `确认` is received, triggering `utils.rewrite_markdown_table()`.
- **Format Sanitization Gotcha:** All agent outputs are sanitized via `utils.sanitize_format(text)`. This function **completely strips out double-asterisk bold tags (`**`)**, converts list bullets to `-`, and structures indentation. **Do not write double-asterisk bold styling (`**`) in agent responses.**

## 📐 Math/LaTeX Formatting Rules
- **Markdown / Sync Context (`sync.py`):** Uses standard `$inline$` and `$$block$$` formulas.
- **Anki Card Context (`prompt.txt`):** Cards generated by agents **MUST** use `\(...\)` (inline) and `\[...\]` (block) for Anki MathJax compatibility. **Using `$` or `$$` is strictly forbidden** within the cards.

## 🔄 Anki-Obsidian Sync Protocol
- **Metadata:** Obsidian notes require a YAML frontmatter `id`. Cards under the `## 卡片` markdown table must have a numeric Anki ID in the third column.
- **Media:** Double-bracket image links (e.g., `[[image.png]]`) must exist in the `Files/` directory under the vault root; `sync.py` uploads them via `storeMediaFile`.
- **Suspension / Revival:** Card suspension in Anki is synced back as a `#auto_suspended` tag inside the question column. To unsuspend/revive, users prepend `#revive` to the question column in Obsidian; the next sync unsuspends the card and automatically cleans up the `#revive` tag.
